第一次：grok 更改需求文档
项目范围完全没边界
问题：没说“花卉限哪几种”“疾病覆盖多少种”。
后果：开发到一半可能突然加“所有球根花卉”，MVP 永久做不完。
没有准确率指标
问题：整个文档没出现“75%”“80%”任何一个数字。
后果：做出来 50% 准确率也能说“能用”，验收永远扯皮。
三层渐进诊断只有名字没内容
问题：“渐进式诊断（三层）”就这一句，没阈值、没流程。
后果：程序员自己脑补，A写成 70% 进下一层，B写成 90%，结果完全两套系统。
RDF/OWL 在 MVP 阶段就是纯坑
问题：直接写“本体用 RDF/OWL”。
后果：MVP 第一周就得学 rdflib + Protégé + SPARQL，进度直接暴死（我们之前踩过无数次了）。
知识库结构零定义
问题：“支持人工/LLM 修改”但没给 JSON 格式。
后果：今天 Claude 加个病是数组，明天加成对象，知识库 3 天就乱成翔。
输出报告没统一格式
问题：只说“输出疾病名称、置信度、治疗方案”，没说 JSON 还是 HTML、字段叫啥。
后果：前端写一套，后端写一套，对不上接口。
二次问诊完全空
问题：写了“支持二次问诊”但没说问啥、怎么问、问几个。
后果：实现出来可能就是“你还有啥症状？”，用户一脸懵。
没有验收标准
问题：全文档没一条“怎么算完成”。
后果：做完一堆代码，你说“不行”，他说是“我按需求做了”，死循环。
